**測試結果：**

- ✅ 無候選品項：正確退回建議品項名稱

#### 2.3 新增函數：parse_item_selection()

**位置：** 第 148-175 行（28 行）

**功能：**
- 解析「編號 + 地點 + 數量」格式
- 例如：`1 大順家 2`、`2 南屏 0.5`、`3 外出用品 1`
- 支援簡稱地點：`1 大順 2` → `(1, "大順家", 2.0)`
- 支援中文數量：`1 大順 半` → `(1, "大順家", 0.5)`
- 支援無空格：`1大順家2` → `(1, "大順家", 2.0)`

**返回：** `(index, location, qty)` 或 `(None, None, None)`

**測試結果：**
- ✅ 標準格式：`1 大順家 2` → `(1, "大順家", 2.0)`
- ✅ 簡稱地點：`2 南屏 0.5` → `(2, "南屏家", 0.5)`
- ✅ 中文數量：`1 大順 半` → `(1, "大順家", 0.5)`
- ✅ 無空格：`2 南屏家1` → `(2, "南屏家", 1.0)`
- ✅ 格式錯誤：正確返回 `(None, None, None)`

#### 2.4 修改函數：handle_photo_message()

**位置：** 第 178-254 行

1. 將 `extract_item_name()` 改為 `extract_items_from_vision()`

   - `len(items) == 1` → single mode（維持原本快速更新流程）
   - `len(items) >= 2` → multi mode（顯示候選清單，要求編號選擇）

**單一品項流程（維持原本）：**

**多品項流程（新增）：**
```python
else:
    pending_items[chat_id] = {
```
